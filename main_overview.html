frontend/main.js (한글, 고1 이해도)

  • 기본 요소: 버튼(발전기/거래/업그레이드/정보), 메인 화면, 유저 이름 DOM을 잡고 contentMode, currentUser로 현재 화면과 로그인 상태를 기억합니다.
  • 로그인/저장 도우미: getStoredUser, persistUser, syncUserState, getAuthToken, getAuthContext로 로컬스토리지에 저장된 유저·토큰을 읽고, 변경 시 UI·스토리지 둘 다 갱신합니다.
  • 고정 데이터: generators(3가지 발전기 기본 정보), upgrades(업그레이드 종류/가격 성장). 서버에서 내려주는 발전기 타입 정보를 generatorTypeMap 등에 넣어 ID·비용을 확인합니다.
  • 유틸: 플레이시간 계산, 업그레이드 비용 계산, 이미지 경로 만들기, HTML 이스케이프, 드래그 툴팁 위치 잡기 등 작은 함수들이 있습니다.
  • 에너지 타이머: 설치된 발전기 생산량 + 생산 보너스를 합산해 1초마다 에너지를 올리고 저장합니다.
  • 화면별 기능
    • 발전기: generatorMode가 드래그 가능한 카드와 툴팁 모달을 렌더링합니다.
    • 거래: tradeMode가 시장 가격을 불러오고 에너지 판매, 공급 업그레이드 버튼을 제공합니다.
    • 업그레이드: 업그레이드 카드 리스트를 만들고, 버튼 클릭 시 서버에 POST 후 상태를 새로고침합니다.
    • 정보: 플레이 시간, 에너지/돈, 랭크를 간단히 보여줍니다.
  • 진행도 저장/불러오기: loadGeneratorTypes로 발전기 타입을 미리 로드하고, saveProgress/loadProgress로 서버에 설치 정보 저장/조회, renderSavedGenerators로 화면에 다시 그립니다.
  • 드롭 설치: initDropHandlers가 드롭 이벤트를 연결합니다. 드롭 시 로그인·비용 확인 → /progress 호출 → 유저 돈/에너지 갱신 → 화면에 아이콘 배치.
  • 시작 흐름: DOMContentLoaded 시 타입 로드 → 저장된 유저 로드 → 화면 렌더 → 드롭 핸들러 등록 → 서버 진행도 재생성.

backend/main.py (한글, 고1 이해도)

  • 환경 설정: DATABASE_URL(기본 sqlite ./data/energy_tycoon.db)을 읽고 폴더를 만들어 둡니다. CORS는 FRONTEND_ORIGINS(기본 *)을 허용합니다.
  • DB 모델: User, GeneratorType, Generator, MapProgress. 오래된 DB에 없을 수 있는 업그레이드 컬럼을 _ensure_user_upgrade_columns가 추가합니다.
  • 시드 데이터: 시작 시 기본 발전기 타입 3개를 한 번만 넣습니다.
  • 토큰/인증: 메모리에 _token_store를 두고, 헤더에서 토큰을 꺼내 require_user로 유저를 찾습니다. (get_user_and_db가 의존성으로 (user, db, token)을 줌)
  • 업그레이드 계산: UPGRADE_CONFIG에 가격/성장률/필드가 정의됨. calculate_upgrade_cost, apply_upgrade가 비용 차감 후 레벨을 올립니다. ensure_same_user로 요청한 유저와 토큰 유저를 비교합니다.
  • 시장 로직: MARKET_STATE에 누적 판매량(sold_energy)과 기본 환율을 저장. current_market_rate가 판매량이 많을수록 가격을 낮추고, supply_bonus로 약간 올려줍니다.
  • 주요 엔드포인트
    • 회원/로그인: POST /signup, POST /login, POST /logout, POST /refresh
    • 거래: POST /change/energy2money(에너지→돈, 시장 환율 적용), POST /change/money2energy(돈→에너지 1:1)
    • 시장 조회: GET /market (현재 환율/누적 판매량)
    • 랭크: GET /rank, GET /ranks
    • 진행도: GET /progress(설치된 발전기 목록), POST /progress(돈 차감 후 발전기 설치 저장)
    • 업그레이드: POST /upgrade/production, /heat_reduction, /tolerance, /max_generators, /supply
  • 주의 사항
    • 토큰은 서버 메모리에만 있어 재시작하면 로그인 정보가 사라집니다.
    • sqlite 컬럼 추가는 “필요 시 추가” 방식이라 기존 DB를 덮지 않고 확장만 합니다.
