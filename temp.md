# 1. 프로젝트를 한 이유 및 개발 동기
이 프로젝트는 "Energy Tycoon"이라는 제목으로, 유휴 상태의 게임 플레이를 통해 에너지 생산 및 관리의 재미를 제공하고자 기획되었습니다. 사용자들이 간단한 조작으로 복잡한 경제 시스템을 경험하고, 자신만의 에너지 제국을 건설하는 만족감을 느낄 수 있도록 하는 것이 목표였습니다.

# 2. 사용한 언어, 기술 스택
- **백엔드**: Python, FastAPI, SQLAlchemy, PostgreSQL (Neon DB)
- **프론트엔드**: React.js, JavaScript, HTML, CSS
- **배포**: Docker, Google Cloud Platform (Cloud Run, Cloud Build)
- **데이터베이스**: Neon DB (PostgreSQL 호환)

# 3. 발생한 문제와 어려웠던 점
1.  **실시간 데이터 동기화**: 유휴 게임의 특성상 클라이언트와 서버 간의 데이터 동기화가 실시간으로 이루어져야 했습니다. 특히 사용자가 오프라인일 때도 에너지 생산이 지속되어야 하는 로직 구현이 복잡했습니다.
2.  **대용량 숫자 처리**: 게임 내에서 다루는 에너지 및 재화의 단위가 기하급수적으로 증가함에 따라, JavaScript의 Number 타입 범위 문제와 백엔드에서의 정확한 계산 처리가 어려웠습니다.
3.  **데이터베이스 스키마 설계 및 마이그레이션**: 게임 데이터의 복잡성(생성기, 업그레이드, 특수 건물 등)으로 인해 효율적인 데이터베이스 스키마 설계가 중요했으며, 개발 과정에서의 스키마 변경 및 마이그레이션 관리가 까다로웠습니다.
4.  **배포 환경 구성**: Docker와 Google Cloud Platform을 이용한 CI/CD 파이프라인 구축 과정에서 다양한 환경 설정 문제와 에러를 마주했습니다.

# 4. 발생한 문제를 어떻게 해결했는지
1.  **실시간 데이터 동기화**: `useGameSync` 훅을 개발하여 주기적으로 클라이언트 데이터를 서버와 동기화하고, `sync_logic.py`에서 오프라인 동안의 생산량을 계산하여 반영하도록 구현했습니다. 이로써 사용자 접속 여부에 관계없이 일관된 게임 경험을 제공할 수 있었습니다.
2.  **대용량 숫자 처리**: 프론트엔드에서는 `bigValue.js` 유틸리티를 사용하여 문자열 기반의 대용량 숫자 연산을 구현했습니다. 백엔드에서는 Python의 `Decimal` 타입을 사용하여 부동 소수점 오류 없이 정확한 계산을 보장했습니다.
3.  **데이터베이스 스키마 설계 및 마이그레이션**: 초기 설계를 신중하게 진행했으며, `init_db.py` 스크립트를 통해 초기 데이터베이스 구성을 자동화했습니다. 또한, Alembic과 같은 마이그레이션 도구를 활용하여 스키마 변경 이력을 관리하고, `migrate_to_neon.py` 스크립트를 통해 특정 환경으로의 마이그레이션을 지원했습니다.
4.  **배포 환경 구성**: `Dockerfile`과 `cloudbuild.yaml`을 세심하게 작성하여 Docker 이미지를 빌드하고 Google Cloud Run에 배포하는 자동화된 CI/CD 파이프라인을 구축했습니다. 에러 발생 시 로그를 면밀히 분석하고 GCP 문서를 참조하여 문제를 해결했습니다.

# 5. 협업한 과정
이 프로젝트는 단독으로 진행되었습니다.
